GitHubで「このエンジニア、どんな人たちとつながってるんだろう」と思ったことはないですか？

普段 GitHub のフォロワー欄を眺めていても、テキストの一覧が並ぶだけで**関係の全体像**は見えてきません。「AさんとBさんって相互フォローなのかな」「自分のフォロワー同士はどれくらいつながってるんだろう」──そんな疑問を視覚的に解決したくて、**GraphHub** というWebアプリを作りました。

## 作った背景

きっかけは「GitHub のフォロー関係を整理したい」という個人的な欲求でした。

フォロー数が増えてくると「この人誰だっけ」「なんで相互じゃないんだろう」といったことが気になり始めます。でも GitHub の標準UIだと、フォロワー・フォロー中のページを行ったり来たりするしかなく、全体像が全然見えない。

グラフ理論的に見れば、フォロー関係は「有向グラフ」そのものです。だったら**グラフとして描画してしまえばいい**と思って作りました。

## 技術スタック

```
Next.js 14 (App Router)
TypeScript 5 (strict mode)
Tailwind CSS
NextAuth.js v5 (GitHub OAuth)
tRPC
vis-network / force-graph（グラフ描画）
Zustand（状態管理）
Vitest + React Testing Library（テスト）
Vercel（デプロイ）
```

モダンなT3スタック的な構成で、型安全を徹底しています。特に **tRPC** を使うことで、フロントエンドからGitHub APIを呼ぶ処理を型安全に書けているのが気に入っています。

## ちょっとした使い道

せっかくなので、実際に役立つ使い方をいくつか：

**フォロワー整理に**
誰と相互フォローで、誰がフォローしてくれているだけかが一目でわかるので、フォロー整理の参考になります。

**気になるエンジニアのネットワーク探索に**
「このOSSのメンテナーはどんな人とつながってるんだろう」と思ったとき、その人のグラフを見ると同じ分野で活躍している人たちが芋づる式に見つかります。

**勉強会・コミュニティの構造把握に**
特定コミュニティのオーガナイザーのグラフを見ると、そのコミュニティの中心メンバーがひと目でわかったりします。
