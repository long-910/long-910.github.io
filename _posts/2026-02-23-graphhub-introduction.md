GitHubで「このエンジニア、どんな人たちとつながってるんだろう」と思ったことはないですか？

普段 GitHub のフォロワー欄を眺めていても、テキストの一覧が並ぶだけで**関係の全体像**は見えてきません。「AさんとBさんって相互フォローなのかな」「自分のフォロワー同士はどれくらいつながってるんだろう」──そんな疑問を視覚的に解決したくて、**GraphHub** というWebアプリを作りました。

https://github.com/long-910/GraphHub

---

## 何ができるの？

一言で言うと、**GitHubのフォロー/フォロワー関係をネットワークグラフとして描画するWebアプリ**です。

![GraphHub デモ](https://media.githubusercontent.com/media/long-910/GraphHub/main/docs/demo.gif)

ユーザー名を入力するだけで、そのユーザーを中心としたフォロー関係のネットワークが広がります。

### 色分けされた関係性

グラフの各ノード（人）は関係の種類によって色が異なります：

| 色          | 意味                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| 🟣 パープル | **相互フォロー**（自分もフォロー・相手もフォロー）                        |
| 🟢 グリーン | **片方向フォロー（自分→相手）**（フォローしてるけど返ってきてない）       |
| 🔵 ブルー   | **片方向フォロー（相手→自分）**（フォロワーだけど自分はフォローしてない） |

「あの人、相互フォローのつもりだったのに実は返ってきてなかった…」みたいな発見が意外とあったりします。

### アバター付きノード

各ノードにはGitHubのプロフィール画像が表示されます。ユーザー名だけの一覧より、アイコンで顔を覚えているエンジニアはすぐ見つけられます。

### ノードをクリックして連鎖的に探索できる

どのノードをクリックしても、**そのユーザーのグラフに切り替わります**。芋づる式にネットワークを掘り下げていけるので、気づくと知らない面白いエンジニアを発見している──という体験ができます。

_↑ デモ（上のGIF）でもノードのクリックによる画面遷移と、ズーム・パン操作が確認できます_

### フィルター表示

- すべての関係を表示
- 相互フォローだけ表示
- 片方向フォローだけ表示

といった絞り込みができるので、「本当に仲の良いつながり」だけを見たいときに便利です。

### エクスポート機能

作成したグラフは以下の形式でダウンロードできます：

- **PNG** – 画像として保存・共有用
- **JSON** – データ解析用
- **CSV** – スプレッドシートで加工したい場合

## 作った背景

きっかけは「GitHub のフォロー関係を整理したい」という個人的な欲求でした。

フォロー数が増えてくると「この人誰だっけ」「なんで相互じゃないんだろう」といったことが気になり始めます。でも GitHub の標準UIだと、フォロワー・フォロー中のページを行ったり来たりするしかなく、全体像が全然見えない。

グラフ理論的に見れば、フォロー関係は「有向グラフ」そのものです。だったら**グラフとして描画してしまえばいい**と思って作りました。

---

## 面白いところ・こだわりポイント

### ネットワーク探索の快感

グラフをぐりぐりズームして、気になるノードをクリックしてまたそこからネットワークが広がる──この連鎖的な探索体験が個人的にお気に入りです。

有名なOSSコントリビューターを中心に描画すると、巨大な相互フォローのクラスターが現れたりして、GitHubの「コミュニティ構造」が見えてきます。

### GitHub OAuth でAPI制限を回避

GitHub APIは未認証だと **60リクエスト/時間** という制限があります。GraphHub はGitHub OAuthでログインすることで **5,000リクエスト/時間** まで引き上げています。

大きなネットワークを描画するのにこの差は結構大きく、ログインするかどうかで体験が全然変わります。

### force-directed graph の自然な配置

ノードの配置はforce-directed graphアルゴリズムで自動的に決まります。物理シミュレーションのように引き合いと反発でノードが動いて落ち着く様子はそれだけでちょっと楽しいです。

## 技術スタック

```
Next.js 14 (App Router)
TypeScript 5 (strict mode)
Tailwind CSS
NextAuth.js v5 (GitHub OAuth)
tRPC
vis-network / force-graph（グラフ描画）
Zustand（状態管理）
Vitest + React Testing Library（テスト）
Vercel（デプロイ）
```

モダンなT3スタック的な構成で、型安全を徹底しています。特に **tRPC** を使うことで、フロントエンドからGitHub APIを呼ぶ処理を型安全に書けているのが気に入っています。

## 使い方・セルフホスト

### 前提

- Node.js 18.17 以上
- GitHub アカウント（OAuth アプリの作成が必要）

### 1. GitHubでOAuthアプリを作成

[GitHub Developer Settings](https://github.com/settings/developers) → **OAuth Apps** → **New OAuth App** から作成します。

| 項目                       | 値                                               |
| -------------------------- | ------------------------------------------------ |
| Application name           | GraphHub（任意）                                 |
| Homepage URL               | `http://localhost:3000`                          |
| Authorization callback URL | `http://localhost:3000/api/auth/callback/github` |

作成後に表示される **Client ID** と **Client Secret** をメモします。

### 2. リポジトリをクローン

```bash
git clone https://github.com/long-910/GraphHub.git
cd GraphHub
npm install
```

### 3. 環境変数の設定

```bash
cp .env.example .env
```

`.env` を以下のように編集：

```env
GITHUB_CLIENT_ID=（取得したClient ID）
GITHUB_CLIENT_SECRET=（取得したClient Secret）
AUTH_SECRET=（以下のコマンドで生成）
NEXTAUTH_URL=http://localhost:3000
```

`AUTH_SECRET` の生成：

```bash
npx auth secret
```

### 4. 起動

```bash
npm run dev
```

`http://localhost:3000` にアクセスして、GitHub でログインするだけです。

あとは検索フォームにGitHubのユーザー名を入れると、グラフが表示されます。まず自分のユーザー名で試してみてください。

## ちょっとした使い道

せっかくなので、実際に役立つ使い方をいくつか：

**フォロワー整理に**
誰と相互フォローで、誰がフォローしてくれているだけかが一目でわかるので、フォロー整理の参考になります。

**気になるエンジニアのネットワーク探索に**
「このOSSのメンテナーはどんな人とつながってるんだろう」と思ったとき、その人のグラフを見ると同じ分野で活躍している人たちが芋づる式に見つかります。

**勉強会・コミュニティの構造把握に**
特定コミュニティのオーガナイザーのグラフを見ると、そのコミュニティの中心メンバーがひと目でわかったりします。
