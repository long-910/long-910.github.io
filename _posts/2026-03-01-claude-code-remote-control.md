「デスクで始めたコーディング作業を、ソファに移動してスマホで続けたい」──Claude Code の **Remote Control（リモートコントロール）** はまさにそれを実現する機能です。

ローカルマシンで動いている Claude Code セッションを、スマートフォン・タブレット・他のPCのブラウザなど **任意のデバイスから継続操作** できます。計算処理はあくまで自分のマシン上で動き続け、画面だけ別デバイスに延長されるイメージです。

---

## Remote Control とは

Remote Control は、**ローカルで起動中の Claude Code セッションにリモートデバイスから接続できる** 機能です。

重要なのは「**ローカル実行のまま**」という点です。同じような機能に見える「Claude Code on the web」（クラウドで動く）とは根本的に異なります。

```
通常の Claude Code:
[ターミナル] ←→ [ローカルマシンで実行]

Remote Control:
[ターミナル]  ←→ [ローカルマシンで実行] ←→ [スマホ・タブレット・別PC]
[スマホ]      ↗                           ↘ [claude.ai/code]
```

### Remote Control と「Claude Code on the web」の違い

| 比較項目 | Remote Control | Claude Code on the web |
|---|---|---|
| **実行場所** | 自分のローカルマシン | Anthropicのクラウド |
| **ファイルアクセス** | ローカルファイルにフルアクセス | クラウド上のみ |
| **MCP・ツール** | ローカル設定がそのまま使える | クラウド環境依存 |
| **プライバシー** | ソースコードがクラウドに上がらない | クラウド経由 |
| **利用開始** | ローカルマシンの起動が必要 | どこからでもすぐ開始可能 |
| **向いている用途** | 機密プロジェクト・既存の作業の継続 | スポット作業・マシンなし時 |

プライバシーを重視する開発や、ローカルMCPサーバーを活用した作業には Remote Control が向いています。

## 使えるプラン

2026年3月時点では以下の通りです：

| プラン | Remote Control |
|---|---|
| **Max プラン** | ✅ 利用可能 |
| **Pro プラン** | 近日対応予定 |
| **API キーのみ** | ❌ 非対応（claude.ai でのログインが必要） |
| **Teams / Enterprise** | ❌ 非対応 |

:::message
Remote Control は `claude login` で claude.ai アカウントに認証している状態が必要です。API キーのみでの認証には対応していません。
:::

## セッションの起動方法

### 方法1：コマンドラインから直接起動

```bash
cd ~/my-project
claude remote-control
```

### 方法2：既存セッション内から起動

Claude Code のチャット画面で以下のいずれかを入力：

```
/remote-control
```

または短縮形：

```
/rc
```

### 方法3：すべてのセッションで自動有効化

毎回コマンドを打つのが面倒な場合は、設定で常時有効にできます。

```
/config
```

設定メニューから「Enable Remote Control for all sessions」を `true` にセットすると、起動するたびに自動でリモートアクセスが有効になります。

## リモートデバイスからの接続方法

Remote Control を起動すると、ターミナルにセッション情報が表示されます。

```
✓ Remote Control 有効化
  セッション URL: https://claude.ai/code?session=xxxxxxxx
  セッション名: my-project（リネームで変更可）

  QRコードを表示するには Space キーを押してください
```

### 接続方法は3つ

**1. URL を直接開く**

表示された URL をブラウザで開くだけです。任意のデバイスから claude.ai/code にアクセスして貼り付けても接続できます。

**2. QR コードをスキャン**

スペースキーを押すと QR コードが表示されます。スマートフォンのカメラで読み取るだけで接続できるので最も手軽です。

```
スペースキーを押すと：

█████████████████████████
██ ▄▄▄▄▄ █▄ ▀▄██ ▄▄▄▄▄ ██
██ █   █ █▄▄▀  █ █   █ ██
...（QR コード）...
```

**3. セッション一覧から選ぶ**

claude.ai/code または Claude スマートフォンアプリを開くと、アクティブなセッション一覧が表示されます。セッション名で識別できるので、事前に `/rename` でわかりやすい名前をつけておくと便利です。

---

## セッション名のつけ方（推奨）

Remote Control を使う前に、セッションに名前をつけておくと管理しやすくなります。

```
/rename "auth-feature-debugging"
/remote-control
```

複数のプロジェクトを並行して進めている場合、セッション一覧で「どれが何の作業か」がひと目でわかります。

## 実際の使用シナリオ

### シナリオ1：デスクからソファへの移動

```bash
# デスクで Claude Code を起動・Remote Control を有効化
cd ~/work/my-app
/rc

# QR コードをスキャンしてスマホに接続
# → ソファに移動してスマホから会話を続ける
# → 複雑なコード変更はスマホから指示するだけ
```

### シナリオ2：外出先からの作業

```bash
# 自宅マシンで Claude Code を事前起動
claude remote-control

# 外出先のタブレットやスマホから
# → claude.ai を開いてセッションに接続
# → 自宅マシンのローカルファイルを参照しながら作業継続
```

### シナリオ3：会議中のメモや指示

```
# デスクトップのターミナルで claude が動いている状態で
# → スマホから会議の内容を貼り付けてコード修正を依頼
# → 会議の合間に確認・修正を繰り返す
```

### シナリオ4：マルチデバイスでの確認

```
ターミナル → コマンドや詳細ログの確認
ブラウザ   → コードレビューや長文の読み取り
スマホ     → 外出時・移動時のクイックチェック

※ 3つのデバイスで同じセッションを参照可能（書き込みは1デバイスずつ）
```

## 詳細オプションと設定

### コマンドラインオプション

```bash
# 詳細ログを表示しながら起動
claude remote-control --verbose

# サンドボックスを有効化してファイル・ネットワークアクセスを制限
claude remote-control --sandbox

# サンドボックスを明示的に無効化
claude remote-control --no-sandbox
```

### 設定ファイルで常時有効化

`~/.claude/settings.json` に設定を追加：

```json
{
  "remoteControl": {
    "enabled": true
  }
}
```

または Claude Code の設定コマンドを使用：

```
/config
→ Remote Control → Enable for all sessions: true
```

## 仕組みと通信の安全性

「リモートからアクセスできる」と聞くと、ポートを開放するのか、セキュリティは大丈夫なのかと気になる方もいるでしょう。

### 通信の仕組み

Remote Control は**自分のマシンからの外向き HTTPS 通信のみ**を使います。ファイアウォールにポートを開けたり、VPN を設定したりする必要はありません。

```
ローカルマシン →（HTTPS アウトバウンドのみ）→ Anthropic API
リモートデバイス →（HTTPS）→ claude.ai → Anthropic API ← ローカルマシン

ローカルマシンへの直接インバウンド接続: なし
```

### セキュリティの仕組み

- すべての通信は TLS（HTTPS）で暗号化
- 接続に使う認証情報は短命で、単一用途（別目的には使えない）
- 各認証情報が独立して期限切れになる設計
- Anthropic の API を経由する標準的な通信のみ

**セキュリティ面でのリスクが低い理由**：ローカルマシンへのインバウンド接続が一切ないため、ポートスキャンや直接攻撃の対象になりません。

## 注意点と現在の制限

### 接続数の制限

1つの Claude Code セッションに同時接続できるリモートデバイスは **1台のみ** です。複数のデバイスから同時に書き込もうとすると、最後に接続したデバイスが優先されます（読み取りのみであれば複数可）。

### マシンの起動が必要

Remote Control は**ローカルプロセスとして動いている**ため、ターミナルを閉じると接続できなくなります。外出先から使いたい場合は、ターミナルを開いたまま席を離れる必要があります。

:::message alert
ノートPCのスリープ設定に注意してください。スリープ中はネットワーク接続が切れるため、Remote Control も一時的に切断されます。スリープから復帰すると自動的に再接続します。
:::

### 長時間のネットワーク切断

ローカルマシンのネットワークが約 **10分以上** 切断されると、セッションがタイムアウトしてプロセスが終了します。

### プラン制限

現時点では Max プランのみ対応です。Pro プランへの展開は近日予定とされています。

```
[PC画面]: 別の開発作業を進める

```

## よくある疑問

### Q: 他のクラウドAIサービスとの違いは？

**A:** Google AI Studio や ChatGPT はすべてクラウドで処理します。Remote Control はコードの処理が**自分のマシンのまま**なので、社内規定でソースコードをクラウドにアップできない場合でも使えます（ただし会話のやり取りは Anthropic API を経由します）。

### Q: スマホからコードを書けるの？

**A:** テキスト入力によるコードの依頼・確認はできます。ただし、スマホのブラウザや Claude アプリは PC のターミナルほどコードの入力には向いていません。「スマホから指示して、変更はローカルで適用される」という使い方が基本です。

### Q: 同じプロジェクトで複数セッションを開いたら？

**A:** 複数の Claude Code セッションをそれぞれ Remote Control で有効化することはできます。セッション一覧に複数表示されるので、`/rename` で名前をつけて区別してください。

### Q: Remote Control 中に通常のターミナル操作はできる？

**A:** Remote Control はバックグラウンドで動くわけではなく、Claude Code のセッション自体として動いています。ターミナルを別タブや別ウィンドウで開いて他の作業をする、という形になります。

## まとめ

| ポイント | 内容 |
|---|---|
| **何ができる** | ローカルの Claude Code セッションをスマホ・タブレット・他PCから操作 |
| **実行場所** | 常にローカルマシン（クラウドに移らない） |
| **起動方法** | `claude remote-control` または `/rc` |
| **接続方法** | URL・QRコード・セッション一覧の3通り |
| **対応プラン** | Max プラン（Pro は近日予定） |
| **セキュリティ** | アウトバウンドHTTPSのみ・ポート開放不要 |
| **主な用途** | 移動中の作業継続・長時間タスクの監視・マルチデバイス活用 |

「デスクにいなくても Claude Code の作業が続けられる」という体験は、一度試すと手放せなくなります。特に長時間かかるリファクタリングやコードレビューを Claude に任せながら、スマホで進捗を確認するスタイルは生産性が大きく変わります。

Max プランを使っている方は、ぜひ `/rc` を試してみてください。
