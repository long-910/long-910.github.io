## はじめに

AIコーディングアシスタントが急速に普及する中、2026年に注目を集めているOSS（オープンソース）ツールが **OpenCode** です。GitHub Copilot、Claude Code、Cursorなど多くの競合がある中で、OpenCodeはどのような特徴を持ち、どこが優れているのでしょうか？

本記事では、初心者の方にもわかりやすいように、OpenCodeの概要・インストール方法・無料で使える範囲・有料APIの使い方・Claude Codeとの違いまでを丁寧に解説します。

## OpenCodeとは？

[OpenCode](https://opencode.ai/) は、ターミナル（コマンドライン）上で動作するオープンソースのAIコーディングエージェントです。SST（Serverless Stack）チームが開発し、MITライセンスで公開されています。

### 主な特徴

- **75以上のAIプロバイダーに対応**（Claude、OpenAI、Gemini、ローカルLLMなど）
- **TUI（ターミナルUI）** による快適な操作性
- **LSP（Language Server Protocol）統合** でコードの意味的な理解
- **MCP（Model Context Protocol）対応** でツール拡張が可能
- **完全オープンソース**（MIT License）
- ターミナル、IDE拡張、デスクトップアプリとして利用可能

2026年3月時点で、GitHubスター数は112,000以上、月間利用者は250万人以上を誇ります。

## インストール方法

OpenCodeはさまざまな方法でインストールできます。

### curl（Linux/macOS 推奨）

```bash
curl -fsSL https://opencode.ai/install | bash
```

### npm

```bash
npm install -g opencode-ai
```

### bun

```bash
bun add -g opencode-ai
```

### Homebrew（macOS/Linux）

```bash
brew install anomalyco/tap/opencode
```

### Arch Linux

```bash
paru -S opencode
```

### Windows

```bash
# Scoop
scoop install opencode

# Chocolatey
choco install opencode
```

### Nix / NixOS

```bash
nix run nixpkgs#opencode
```

インストール後は、ターミナルで `opencode` と入力するだけで起動します。

## 初回セットアップ

起動すると、AIプロバイダーを選択する対話型のプロンプトが表示されます。

```bash
opencode
```

セットアップ手順：
1. `/connect` コマンドを入力
2. 使いたいAIプロバイダーを選択（Anthropic、OpenAI、Google など）
3. APIキーを入力
4. `/models` コマンドでモデルを選択

環境変数（`ANTHROPIC_API_KEY`、`OPENAI_API_KEY` など）やプロジェクトの `.env` ファイルからAPIキーを自動検出する機能もあります。

## 無料で使える範囲

OpenCode自体は**完全無料のオープンソースソフトウェア**です。ただし、AIモデルの利用には費用がかかる場合があります。

### 無料で使えるもの

| 内容 | 詳細 |
|------|------|
| OpenCodeツール本体 | 無料（OSSのため） |
| ローカルLLM（Ollama/LM Studio） | 無料（自分のPCで動かす） |
| 一部の期間限定無料モデル | xAI Grok Code Fast 1、GLM 4.7、MiniMax M2.1 など |
| GitHub Copilot連携 | Copilotサブスクリプションがあれば追加費用なし |

### 無料モデルの注意点

期間限定の無料モデルにはいくつかの制約があります：

- **品質が低め**：無料モデルは高性能なコーディングモデルではない場合が多い
- **データ使用**：フィードバック収集期間中はデータがモデル学習に使われる可能性がある
- **トークン消費が速い**：無料トークンを一瞬で使い切る報告もある

```
⚠️ 無料モデルは「試用期間中のフィードバック収集目的」のものが多く、
   期間終了後は有料になる可能性があります。
```

## 料金プラン（2026年3月時点）

OpenCodeには、無料のOSSコア以外にいくつかのオプションがあります。

### OpenCode Zen（従量課金）

- **ベータ版**として提供中
- $20のデポジットで開始
- 使った分だけ支払う**従量課金制**
- OpenCodeチームがテスト・最適化済みのモデルを提供
- **ゼロデータ保持ポリシー**（モデル学習に使われない）
- 現時点ではワークスペース機能は無料
- 価格はコストの上乗せなし（処理手数料のみ）

### OpenCode Go（$10/月）

- 月額$10の定額サブスクリプション
- 人気のオープンモデルに安定したアクセスが可能
- 使用量は「ドル換算」で管理（安いモデルなら多く使える）
- 上限に達した後はZenの残高から自動的に補填
- 主に国際ユーザー向けに最適化

### OpenCode Black（$200/月・現在一時停止中）

- Claude、GPT、Gemini、GLM-4.7、Kimi K2 などのトップモデルに対応
- 2026年初頭に限定公開されたが、現在は新規受付が一時停止中
- [opencode.ai/black](https://opencode.ai/black) で詳細確認可能

### 自分のAPIキーを使う（BYOK：Bring Your Own Key）

- AnthropicやOpenAIなどのAPIキーをそのまま利用できる
- Zenと同時に使うことも可能
- トークンはプロバイダーに直接課金される

## 有料APIの設定方法と注意点

### Anthropic（Claude）APIの設定

```bash
# 方法1：コマンドで設定
# OpenCode起動後に /connect → Anthropic を選択し、APIキーを入力

# 方法2：環境変数で設定
export ANTHROPIC_API_KEY="your-api-key-here"

# 方法3：.envファイルに記載（プロジェクトルート）
echo "ANTHROPIC_API_KEY=your-api-key-here" > .env
```

#### Anthropic API利用の注意点

:::message alert
**⚠️ 重要：Claude Pro/MaxサブスクリプションはOpenCodeで使用不可**

2026年1月9日、AnthropicはClaude Pro/MaxサブスクリプションのOAuthトークンをOpenCodeを含む**サードパーティツールで使用することを禁止**しました。

Anthropicの利用規約（2026年2月19日改定）には、Free・Pro・Maxプランで発行されたOAuthトークンをサードパーティツールで使用することを**明確に禁止**する条項が追加されました。違反するとアカウントが停止される可能性があります。

**OpenCodeでClaudeを使う場合は、必ずAnthropicのAPIキー（従量課金）を使ってください。**
:::

- APIキーは [console.anthropic.com](https://console.anthropic.com) で発行
- APIの料金は**トークン数に応じた従量課金**（モデルによって異なる）
- `claude-sonnet-4-5` は性能とコストのバランスが良くおすすめ

### OpenAI APIの設定

```bash
# 方法1：コマンドで設定
# OpenCode起動後に /connect → OpenAI を選択し、APIキーを入力

# 方法2：環境変数で設定
export OPENAI_API_KEY="your-api-key-here"
```

OpenAI APIキーは [platform.openai.com/api-keys](https://platform.openai.com/api-keys) で発行できます。

### opencode.json での設定（複数プロバイダー）

プロジェクトのルートに `opencode.json` を作成することで、複数のプロバイダーを管理できます：

```json
{
  "providers": {
    "anthropic": {
      "apiKey": "{env:ANTHROPIC_API_KEY}"
    },
    "openai": {
      "apiKey": "{env:OPENAI_API_KEY}"
    },
    "groq": {
      "apiKey": "{env:GROQ_API_KEY}"
    }
  },
  "agents": {
    "coder": {
      "model": "anthropic/claude-sonnet-4-5",
      "maxTokens": 5000
    }
  }
}
```

`{env:変数名}` という記法を使うと、環境変数を設定ファイル内で参照できます。APIキーを設定ファイルに直書きしなくて済むので安全です。

設定ファイルの読み込み優先順位（後のものが優先）：
1. リモート設定（`.well-known/opencode`）
2. グローバル設定（`~/.config/opencode/opencode.json`）
3. 環境変数 `OPENCODE_CONFIG` で指定したファイル
4. プロジェクト設定（プロジェクトルートの `opencode.json`）

:::message alert
**セキュリティの注意：** `opencode.json` にAPIキーを直接書いた場合は、`.gitignore` に追加してリポジトリにコミットしないようにしましょう。`{env:変数名}` 記法を使うのがベストプラクティスです。
:::

## 主な機能の詳細

### 1. LSP統合（コード理解の深化）

OpenCodeの大きな特徴の一つが、LSP（Language Server Protocol）との統合です。

- 起動時にプロジェクトの言語を自動検出
- 対応言語：TypeScript、Python（PyRight）、Rust、Swift、Terraform など
- AIがコードベースを意味的に理解してより正確なサポートを提供
- VS Codeと同様の「定義へ移動」「補完」機能をAIが活用

### 2. MCP（Model Context Protocol）サポート

```json
{
  "mcp": {
    "servers": {
      "filesystem": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
      }
    }
  }
}
```

Claude Codeで使っているMCPサーバーをそのまま流用できます。

### 3. マルチセッション・コラボレーション

- 同じプロジェクトで複数のAIエージェントを並行して実行可能
- セッション共有リンクを生成してチームメンバーと共有
- 共有後は非共有にすることも可能

### 4. カスタムエージェント・コマンド

よく使うプロンプトをMarkdownファイルとして保存して再利用できます。エージェントは `.opencode/agents/` ディレクトリに保存します：

```markdown
<!-- .opencode/agents/reviewer.md -->
---
description: コードレビューエージェント
model: anthropic/claude-sonnet-4-5
---
このコードをレビューして、以下の観点でフィードバックをください：
1. バグの可能性
2. セキュリティの問題
3. パフォーマンスの改善点
```

### 5. GitHub連携

GitHubのPRコメントに `/opencode` または `/oc` とメンションするだけで、GitHub Actions上でタスクを実行できます。

### 6. IDEとの統合

- **VS Code / Cursor / Windsurf / VSCodium**：`Cmd+Esc`（Mac）または `Ctrl+Esc`（Windows/Linux）で分割ターミナルで起動
- **JetBrains IDE、Zed、Neovim、Emacs** など：ACP（Agent Client Protocol）経由でサポート

### どちらを選ぶべき？

**OpenCodeが向いている人**

- コスト重視で複数のAIプロバイダーを使い分けたい
- Claudeだけでなく、OpenAI・Geminiも試してみたい
- ローカルLLMでプライバシーを重視したい
- オープンソースで中身を把握・カスタマイズしたい
- GitHub CopilotやGroqなど既存のサブスクリプションを活用したい
- 複数のモデルを組み合わせてコスト最適化したい

**Claude Codeが向いている人**

- Claude（特にClaude Opus）の最高性能を追求したい
- Anthropicの最新機能に即時アクセスしたい
- セットアップを最小限に抑えてシンプルに使いたい
- AnthropicのMax/Proプランを契約していて追加費用を避けたい
- Anthropicエコシステムとの深い統合が必要

:::message
**ポイント：** Claude CodeはAnthropicのモデルを使うなら最高の体験を提供しますが、OpenCodeは「モデルのベンダーロックインを避けたい」という場合に最善の選択肢です。
:::

### 起動してコードを書く

```bash
# プロジェクトのディレクトリに移動
cd my-project
```

## 注意点まとめ（ハマりやすいポイント）

### ⚠️ Anthropic OAuth 問題（最重要）

2026年1月以降、Claude Pro/Maxサブスクリプションをサードパーティツールで使うことはAnthropicの利用規約違反です。**アカウント停止のリスク**があるため、必ずAPIキー（従量課金）を使用してください。

### ⚠️ モデルID の書式

モデルを指定する際は `プロバイダーID/モデルID` の形式で記述します：

```bash
# 正しい例
opencode --model anthropic/claude-sonnet-4-5

# 間違い例
opencode --model claude-sonnet-4-5
```

### ⚠️ カスタムエンドポイント（baseURL）

OpenAI互換APIのカスタムエンドポイントを使う場合、`baseURL` は `/v1` で終わる必要があります：

```json
{
  "providers": {
    "custom": {
      "baseURL": "https://your-endpoint.example.com/v1"
    }
  }
}
```

`/v1/chat/completions` まで書くとエラーになります。

### ⚠️ 無料モデルのデータ利用

Big Pickle や MiniMax M2.5 Free などの期間限定無料モデルは、フィードバック収集期間中に入力データがモデル学習に使われる可能性があります。機密情報を含むコードには使用しないようにしましょう。

## よくある質問（FAQ）

### Q. 完全無料で使えますか？

**A.** OpenCodeツール自体は無料です。ローカルLLM（Ollama等）を使えば追加費用ゼロで使えます。ただし、Claude・GPT・Geminiなどのクラウドモデルを使う場合はAPIの利用料金がかかります。

### Q. Claude Proのサブスクリプションでも使えますか？

**A.** **使用できません（利用規約違反）。** 2026年1月9日より、AnthropicはClaude Pro/MaxサブスクリプションのOAuthトークンをサードパーティツールで使用することを禁止しました。違反するとアカウントが停止される可能性があります。OpenCodeでClaudeを使うには、必ずAnthropicのAPIキーを使用してください。

### Q. APIキーが漏れると危険ですか？

**A.** はい、APIキーが漏れると第三者に不正使用される可能性があります。以下の対策を取りましょう：
- `opencode.json` に直書きしない
- 環境変数または `.env`（`.gitignore`に追加済み）を使う
- APIキーに使用量制限・アラートを設定する

### Q. Windows でも使えますか？

**A.** WSL（Windows Subsystem for Linux）経由で使用できます。ネイティブWindowsサポートは限定的です。

### Q. Claude Codeと同時に使えますか？

**A.** はい。それぞれ独立したツールなので、プロジェクトによって使い分けることができます。
